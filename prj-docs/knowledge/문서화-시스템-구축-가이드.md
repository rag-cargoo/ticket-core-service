# 📚 문서화 시스템 구축 및 운영 가이드 (Docsify + GitHub Pages)

> **Purpose**: Markdown으로 작성된 프로젝트 문서를 별도의 빌드 과정 없이 웹사이트 형태로 서비스하기 위한 구축 방법과 운영 규칙을 정리함.
> **Target**: 이 프로젝트의 문서를 관리하거나, 새로운 프로젝트에 동일한 시스템을 구축하려는 개발자.

## 1. 시스템 개요
이 프로젝트는 **GitHub Pages**와 **Docsify**를 사용하여 정적 문서 사이트를 운영합니다.
*   **URL**: [https://rag-cargoo.github.io/2602/](https://rag-cargoo.github.io/2602/)
*   **특징**: 마크다운 파일(`.md`)을 수정하고 푸시하면 즉시 웹사이트에 반영됨 (No Build Process).

---

## 2. 구축 방법 (Setup Guide)
처음부터 이 시스템을 구축하려면 아래 단계를 순서대로 따라 하세요.

### Step 1: 필수 파일 생성
프로젝트 최상위 루트(`/`)에 다음 3가지 파일을 생성합니다.

**1. `index.html` (Docsify 진입점)**
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Tech Docs</title>
  <link rel="stylesheet" href="//cdn.jsdelivr.net/npm/docsify@4/lib/themes/vue.css">
</head>
<body>
  <div id="app"></div>
  <script>
    window.$docsify = {
      name: 'Tech Docs',
      repo: 'https://github.com/사용자명/레포지토리명',
      loadSidebar: 'sidebar-manifest.md', // 사이드바 파일 지정
      subMaxLevel: 2
    }
  </script>
  <script src="//cdn.jsdelivr.net/npm/docsify@4"></script>
</body>
</html>
```

**2. `.nojekyll` (중요!)**
*   내용이 없는 빈 파일을 만듭니다.
*   **역할**: GitHub Pages가 Jekyll 빌드를 시도하다가 `_`로 시작하는 파일이나 폴더를 무시하는 것을 방지합니다.

**3. `sidebar-manifest.md` (사이드바 메뉴)**
*   왼쪽 메뉴를 구성하는 파일입니다.
*   **주의**: GitHub 캐시 문제를 피하기 위해 `_sidebar.md` 대신 고유한 이름을 사용하는 것을 권장합니다.
*   **경로 규칙**: 반드시 **상대 경로**를 사용하세요. (앞에 `/` 붙이지 말 것!)
```markdown
* Tech Knowledge
  * [동시성 제어](workspace/apps/backend/docs/concurrency.md)
```

### Step 2: GitHub Pages 설정
1.  GitHub 리포지토리의 **Settings -> Pages** 메뉴로 이동.
2.  **Source**: `Deploy from a branch` 선택.
3.  **Branch**: `main` (또는 master) 선택.
4.  **Folder**: **`/ (root)`** 선택.
5.  **Save** 버튼 클릭.

## 3. 운영 규칙 (Maintenance Guide)

### 문서를 새로 추가할 때
1.  마크다운 파일 작성 (예: `new-doc.md`).
2.  `sidebar-manifest.md` 열기.
3.  적절한 카테고리 하위에 링크 추가.
    *   ❌ `[제목](/workspace/docs/new-doc.md)` (절대 경로 금지)
    *   ✅ `[제목](workspace/docs/new-doc.md)` (상대 경로 사용)

### 문서를 수정할 때
*   그냥 파일 내용을 수정하고 커밋/푸시하면 끝입니다.

## 4. 트러블슈팅 (FAQ)

### Q1. 사이드바가 안 나오고 '404'가 떠요.
*   **원인 1**: `.nojekyll` 파일이 없어서 Jekyll이 사이드바 파일을 무시함.
*   **원인 2**: 브라우저 캐시 문제. 파일명을 `sidebar-manifest.md`처럼 유니크하게 바꾸고 `index.html` 설정을 변경하세요.

### Q2. 메뉴를 눌렀는데 404 페이지로 이동해요.
*   **원인**: 링크 경로 맨 앞에 `/`가 붙어 있어서, GitHub Pages의 루트(리포지토리명 제외)로 이동했기 때문입니다.
*   **해결**: `sidebar-manifest.md`에서 경로 앞의 `/`를 제거하여 상대 경로로 만드세요.

### Q3. 한글 파일명이 안 열려요.
*   GitHub Pages는 한글 URL 인코딩 이슈가 발생할 수 있습니다.
*   가능하면 **파일명은 영어(kebab-case)**로 하고, 문서 내부의 제목(`# 제목`)만 한글로 작성하는 것을 권장합니다.
